# PoC: как запрос пользователя превращается в Jellyfin media ID

Дата: 2026-06-27

## 1. Цель
Показать пошаговую цепочку, как запрос вида:
Включи серию сериала Офис, где Майклу насрали на ковер
превращается в конкретный Jellyfin item_id.

## 2. Компоненты PoC
1. Нормализация пользовательского запроса:
- build_search_query_prompt.py

2. Выбор релевантного медиа-объекта из доступной библиотеки:
- jellyfin_build_resolution_prompt.py

3. Итеративный выбор внутри коллекции (если на шаге B выбран collection/boxset):
- jellyfin_nl_to_item_id_e2e.py (встроенный повторный выбор по дочерним элементам)

4. Уточнение сезона и серии для уже выбранного сериала (только если на шаге B is_serial = true):
- build_series_episode_question_prompt.py

5. Резолв в item_id через Jellyfin API:
- для фильмов: резолв по выбранному item_id
- для сериалов: series -> season -> episode

## 3. Вход пользователя
Пользовательский запрос:
Включи серию сериала Офис, где Майклу насрали на ковер

## 4. Шаг A: преобразование action-запроса в search-запрос
Скрипт:
build_search_query_prompt.py

Назначение:
Убрать глаголы действия и оставить поисковый смысл.

Ожидаемый JSON-ответ LLM:
{"search_query":"офис Майкл обосрали на ковер"}

Итог шага:
Получен нормализованный запрос для поиска.

## 5. Шаг B: выбор релевантного медиа-объекта из каталога Jellyfin
Скрипт:
jellyfin_build_resolution_prompt.py

Что делает:
1. Подключается к Jellyfin API.
2. Получает список доступных медиа.
3. Нумерует список в формате:
N: Название
4. Формирует prompt для LLM с требованием вернуть только:
- какой именно объект библиотеки является релевантным,
- является ли он сериалом или нет.

На этом шаге скрипт НЕ должен пытаться определить сезон и серию.

Формат результата шага B должен быть концептуально таким:
{"media_index": number|null, "is_serial": boolean|null}

Фактический результат на PoC:
{"media_index": 33, "is_serial": true}

Интерпретация:
1. media_index 33 = The Office.
2. is_serial = true, значит применим следующий шаг уточнения сезона и серии.
3. Если бы это был фильм, пайплайн мог бы сразу переходить к резолву item_id без шага C.

## 5. Шаг B2: если выбран Collection, делаем повторный выбор внутри нее
Этот шаг выполняется автоматически, когда выбранный объект в Jellyfin имеет тип collection-like.

Что делает:
1. Берет дочерние элементы коллекции через Jellyfin API.
2. Повторно запускает LLM-выбор на этом дочернем списке.
3. Повторяет итеративно, пока не будет выбран конечный объект (movie/series).

Пример (проверен):
1. Запрос: включить гарри поттер и кубок огня
2. Первый выбор: Harry Potter Collection
3. Внутренний выбор: Harry Potter and the Goblet of Fire
4. Результат: item_id 80c0037c177a7b219b2a8fe8e3a42361

## 6. Шаг C: уточнение сезона/серии внутри уже выбранного сериала
Скрипт:
build_series_episode_question_prompt.py

Этот шаг выполняется ТОЛЬКО если на шаге B выбранный объект является сериалом.

Поддерживается два режима:
1. Базовый режим: только вопрос по названию сериала + user_request.
2. Усиленный режим: дополнительно передаются реальные эпизоды из Jellyfin
	(season, episode, title, overview) через флаг use-episode-metadata.

Рекомендованный режим для точности: use-episode-metadata.

Входы шага:
1. series_name = The Office
2. user_request = офис Майкл обосрали на ковер

Сформированный вопрос:
В каком сезоне и в какой серии сериала The Office офис Майкл обосрали на ковер

Фиксированный формат ответа LLM:
{"season": integer|null, "episode": integer|null}

Фактический ответ на PoC:
{"season":2,"episode":14}

## 7. Шаг D: резолв в конкретный Jellyfin item_id
Входы:
1. Сериал: The Office
2. Season: 2
3. Episode: 14

Jellyfin API логика:
1. Найти пользователя (smarttv).
2. Найти сериал The Office и получить series_id.
3. Найти season с IndexNumber = 2 и получить season_id.
4. Найти episode с IndexNumber = 14 и получить episode item_id.

Фактический результат PoC:
1. TITLE: The Carpet
2. ITEM_ID: f29d639111274719d30f575d7f549878
3. DETAILS_URL: http://192.168.0.104:8899/web/#/details?id=f29d639111274719d30f575d7f549878&serverId=675d8887c5664e87bcb8dc8ad9dd32f4

## 8. Итоговая цепочка (коротко)
1. User command -> search_query
2. search_query + Jellyfin catalog -> media_index + is_serial
3. Если выбран collection -> итеративный повтор шага 2 на дочерних элементах
4. Если is_serial=true -> user context (+опционально episode metadata) -> season/episode
5. Если is_serial=false -> прямой резолв выбранного item_id
6. Для сериала: series + season + episode -> Jellyfin item_id

## 9. Где возможны ошибки и fallback
1. LLM выбрала объект, но неясно сериал это или нет:
- повторить шаг B с более жестким форматом ответа.

2. LLM выбрала collection, но внутри не получилось выбрать элемент:
- вернуть needs_clarification или повторить выбор на дочернем списке.

3. LLM выбрала сериал, но сезон/серия еще не определены:
- запускать шаг C в режиме use-episode-metadata.

4. LLM вернула несколько эпизодов:
- спросить пользователя, какой из кандидатов запускать.

5. Эпизод не найден по IndexNumber:
- fallback на поиск по Name/Overview/PremiereDate.

## 10. Что уже доказано этим PoC
1. Нормализация action-запроса работает.
2. Выбор релевантного объекта из локального каталога работает.
3. Если выбрана коллекция, итеративный выбор внутри нее работает.
4. Уточнение season/episode заметно точнее с передачей episode metadata.
5. Резолв в конкретный Jellyfin item_id работает для фильмов и сериалов.
